What FocusFlow Will Do

Track daily and weekly habits, visualize streaks, and surface insights that keep you motivated.

  • Daily checklist with streak tracking
  • Weekly summary with charts
  • Reminders for high-priority habits
  • Simple sharing of progress

Planned Features

  1. Week 1: Express setup, landing page, basic styling
  2. Week 2: Habit form + POST route to store entries
  3. Week 3: Connect to database for persistence
  4. Week 4: Auth & personalized dashboards

Why FocusFlow?

Consistency is easier when the feedback loop is short. FocusFlow makes it effortless to log routines, spot gaps, and celebrate wins every week.

API Testing Links

Quick test links for the Habits CRUD API:

Basic CRUD Operations

  • GET /api/habits - Get all habits
  • GET /api/info - Project information

Note: To test GET /api/habits/:id, POST, PUT, and DELETE operations, use tools like Postman, curl, or the browser console.

Filtering Examples

  • GET /api/habits?filter={"title":"Exercise"} - Filter by title
  • GET /api/habits?filter={"title":{"$regex":".*","$options":"i"}} - Filter with regex

Sorting Examples

  • GET /api/habits?sort={"title":1} - Sort by title ascending
  • GET /api/habits?sort={"created_at":-1} - Sort by created_at descending

Projection Examples

  • GET /api/habits?projection={"title":1,"description":1} - Get only title and description
  • GET /api/habits?projection={"title":1} - Get only title

Combined Examples

  • GET /api/habits?filter={}&sort={"title":1}&projection={"title":1,"description":1} - Combined filter, sort, and projection

Testing POST, PUT, DELETE

Use curl or Postman to test these operations:

# POST - Create a new habit
curl -X POST http://localhost:3000/api/habits \
  -H "Content-Type: application/json" \
  -d '{"title":"Exercise","description":"30 minutes daily"}'

# PUT - Update a habit (replace ID with actual ObjectId)
curl -X PUT http://localhost:3000/api/habits/<ID> \
  -H "Content-Type: application/json" \
  -d '{"title":"Exercise","description":"45 minutes daily"}'

# DELETE - Delete a habit (replace ID with actual ObjectId)
curl -X DELETE http://localhost:3000/api/habits/<ID>
